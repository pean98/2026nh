# 인천논현고 교육과정박람회 웹사이트 기술스택 결정안

## 결론

1차 버전은 정적 웹앱으로 제작한다.

추천 스택:

- 프론트엔드: React + TypeScript + Vite
- 스타일: Tailwind CSS
- 아이콘: lucide-react
- 데이터: 로컬 JSON 파일
- 상태 저장: localStorage
- PDF 출력: 브라우저 기반 PDF 생성 라이브러리
- 배포: GitHub Pages
- 저장소: GitHub public repository

## 왜 이 스택인가

### 무료 운영에 가장 적합

GitHub Pages는 HTML, CSS, JavaScript 정적 파일을 GitHub 저장소에서 바로 호스팅할 수 있다. public repository 기준으로 GitHub Free에서도 사용할 수 있어 학교 안내 사이트의 1차 배포에 적합하다.

### 서버 없이 핵심 기능 구현 가능

이번 사이트의 1차 핵심 기능은 다음과 같다.

- 과목 검색
- 학년/교과군/선택 유형 필터
- 태그 기반 탐색
- 과목 상세 페이지
- 관심 과목 담기
- 과목 비교
- 학점 시뮬레이터
- 상담 질문 작성
- PDF 출력

이 기능들은 대부분 서버 없이 브라우저 안에서 구현할 수 있다. 학생별 데이터를 중앙 DB에 저장하지 않고 각 기기의 localStorage에 저장하면 개인정보 부담도 줄어든다.

### 유지보수가 쉽다

과목 정보는 JSON 파일로 관리한다. 이후 과목명, 학점, 키워드, 추천 계열, 평가 방식 등이 바뀌면 코드 전체를 고치지 않고 데이터 파일만 수정하면 된다.

### 모바일 친화적 UI 제작에 적합

React는 검색, 필터, 장바구니형 선택, 시뮬레이터처럼 상태 변화가 많은 화면에 적합하다. Tailwind CSS는 반응형 UI를 빠르게 만들 수 있어 모바일 우선 개발에 좋다.

## 배포 후보 비교

### GitHub Pages

추천도: 높음

장점:

- 무료 사용 가능
- 정적 웹앱 배포에 적합
- 학교 사이트처럼 공개 안내용 웹사이트에 충분함
- Vite 빌드 결과물을 그대로 배포 가능

주의:

- 서버 기능은 없음
- 학생별 제출/교사용 대시보드 같은 기능은 별도 서비스가 필요함

### Vercel

추천도: 중간

장점:

- React/Vite/Next.js 배포가 매우 편함
- 미리보기 배포가 좋음
- 무료 Hobby 플랜이 있음

주의:

- Hobby 플랜은 개인/비상업 용도 중심이다.
- 팀 협업, 학교 조직 운영, 사용량 증가 시 제한을 확인해야 한다.

### Netlify

추천도: 중간

장점:

- 정적 사이트 배포가 편함
- 폼 기능 등 확장성이 있음

주의:

- 무료 사용량 제한 확인 필요
- GitHub Pages보다 운영 구조가 하나 더 늘어난다.

### Supabase

추천도: 2차 이후 검토

장점:

- 로그인, DB, 파일 저장, 제출 데이터 관리 가능
- 교사용 상담 대시보드 확장에 적합

주의:

- 학생 개인정보와 권한 관리 설계가 필요함
- 무료 플랜 제한이 있어 운영 전 검토 필요
- 1차 버전에는 과함

## 단계별 스택

### 1차 MVP

목표: 박람회 당일 바로 쓸 수 있는 과목 탐색 사이트

- React
- TypeScript
- Vite
- Tailwind CSS
- 로컬 JSON
- localStorage
- GitHub Pages

구현 기능:

- 홈 대시보드
- 고교학점제 가이드
- 선택 과목 백과
- 검색/필터/태그 탐색
- 과목 상세
- PDF 자료실 링크

### 2차

목표: 학생 참여형 설계 도구

추가 기능:

- 관심 과목 담기
- 과목 비교
- 학점 계산기
- 국수영 81학점 제한 경고
- 위계 과목 경고
- 상담 질문 작성
- 학업 계획서 PDF 출력

기술:

- localStorage 유지
- PDF 생성 라이브러리 추가

### 3차

목표: 진로·대입 매칭 및 상담 확장

추가 기능:

- 대학별 권장과목 데이터 관리
- 내 선택 과목과 권장과목 매칭
- 꿈두레 공동교육과정 안내
- 교사용 상담 확인

기술 후보:

- Google Sheets 공개 CSV 또는 JSON
- Supabase
- Firebase
- Google Forms/Sheets 연동

## 최종 권장 결정

지금은 `React + TypeScript + Vite + Tailwind CSS + GitHub Pages`로 시작한다.

데이터는 처음에는 로컬 JSON으로 두고, 학생별 저장은 localStorage로 처리한다. 교사용 제출이나 대학별 권장과목 자동 업데이트가 꼭 필요해지는 시점에만 Supabase 또는 Google Sheets 연동을 검토한다.

이 방식이 무료 운영, 빠른 제작, 개인정보 부담 최소화, 향후 확장성 사이에서 가장 균형이 좋다.
