# 진로 매칭 데이터 사이트 적용 정리안

이 문서는 `대학별_전공이수_핵심과목_링크.md`에 모은 원자료를 바탕으로, 실제 사이트의 `진로 매칭` 화면에 적용하기 쉽게 재구성한 문서입니다.

## 적용 방향

1. `진로 매칭`에는 모든 대학의 모든 모집단위를 그대로 넣지 않는다.
2. 학생이 빠르게 이해할 수 있도록 대표 모집단위 카드 중심으로 보여준다.
3. 대학별 자료 성격이 서로 다르므로 카드 유형을 나눈다.
   - `전공별 권장과목형`: 고려대, 인하대, 서울과학기술대, 충남대, 경북대 일부
   - `계열별 권장방향형`: 건국대, 단국대, 아주대, 충북대
   - `교과영역형`: 동국대
   - `대입 반영 유의사항형`: 경희대
   - `추가 확인 필요형`: 서울대
4. 과목명은 가능한 한 인천논현고 사이트의 과목 백과에 있는 과목명으로 맞춘다.
5. 대학 자료의 `대수`, `미적분Ⅰ`, `확률과 통계`, `영어Ⅰ` 등은 본교 선택과목 카드에 없는 경우가 있으므로 `기초 이수 전제` 또는 `공통/일반 과목` 메모로 처리한다.

## 사이트 데이터 구조 제안

현재 `src/data/admissions.ts`의 `AdmissionRecommendation`은 다음 필드만 가진다.

```ts
interface AdmissionRecommendation {
  id: string;
  university: string;
  department: string;
  track: Track;
  coreSubjects: string[];
  recommendedSubjects: string[];
  note: string;
}
```

대학별 자료 출처와 유의사항을 함께 보여주려면 아래 필드 추가를 권장한다.

```ts
interface AdmissionRecommendation {
  id: string;
  university: string;
  department: string;
  track: Track;
  coreSubjects: string[];
  recommendedSubjects: string[];
  note: string;
  sourceLabel?: string;
  sourceUrl?: string;
  sourceType?: "전공별" | "계열별" | "교과영역" | "대입유의";
}
```

## 공통 안내 문구

진로 매칭 화면 상단 또는 권장과목 카드 위에 다음 안내를 넣는다.

> 대학별 권장과목은 지원 자격이나 합격 보장이 아닙니다. 학생의 진로 탐색과 과목 선택을 돕기 위한 참고 자료이며, 실제 지원 전에는 반드시 해당 대학 입학처의 최신 모집요강과 안내 자료를 확인해야 합니다.

## 구현 우선순위

1. `admissionRecommendations`를 위 `1차 반영 추천 카드` 기준으로 교체 또는 확장한다.
2. 카드에 `sourceLabel`, `sourceUrl`, `sourceType` 필드를 추가한다.
3. `진로 매칭` 화면에서 대학별 카드가 너무 많아지지 않도록 다음 필터를 추가한다.
   - 계열 필터
   - 대학 필터
   - 과목명 검색
4. `경희대 대입 반영 유의사항`은 권장과목 카드와 분리된 안내 박스로 배치한다.
5. `자료실`에 `대학별 권장과목 자료` 링크 묶음을 추가한다.

## 구현 시 주의할 점

- 대학 자료의 권장과목은 `지원 자격`이 아니므로 카드에 반드시 안내 문구를 붙인다.
- 서울대 자료는 PDF 텍스트 추출 신뢰도가 낮았으므로 구체 학과 카드로 쓰기 전 원본 대조가 필요하다.
- 동국대는 `과목명`이 아니라 `교과 영역` 기준이므로, 카드 note에 “동국대 자체 교과 영역 기준”이라고 명시한다.
- 단국대와 아주대는 특정 과목명보다 `일반선택 이후 진로·융합선택`이라는 방향성이 핵심이므로, 카드 note를 상세히 쓰는 방식이 적절하다.
- `대수`, `미적분Ⅰ`, `확률과 통계`처럼 본교 과목 백과의 선택과목 목록에 없는 과목은 칩으로 표시하기보다 설명 문장에 넣는다.
